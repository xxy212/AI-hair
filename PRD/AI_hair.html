01 · MVP 核心定位

产品一句话

面向中国自然卷新手的 AI 卷发护理助手，帮助用户理解自己的发质， 并生成入门护理方案、产品选择建议和使用教程。

第一版成功标准

用户能完整经历一次“上传照片 + 填问卷 + 获得推荐 + 查看教程 + 聊天反馈”的流程， 并觉得这像一个真正的产品，而不是普通聊天机器人。

04 · AI Pipeline

推荐系统思路

采用“图片 + 问卷 + 种子产品库 + LLM 结构化输出”的方式。 AI 负责生成发质画像、产品建议、护理教程和聊天调整。

Vision API LLM API Structured JSON Seed Product DB
A. 照片分析 卷度、毛躁、发量、干枯猜测
↓
B. 发质画像 结合问卷生成用户画像
↓
C. 产品建议 推荐产品类型 + 示例产品
↓
D. 教程生成 洗头日、第二天 refresh、新手避坑
↓
E. 聊天反馈 根据用户追问调整方案

05 · 技术架构

模块 建议技术 职责
前端 Web App Next.js / React / Tailwind CSS 首页、上传、问卷、结果页、聊天页。
后端 API Next.js API Routes 或 FastAPI 接收用户输入，调用 AI API，返回结构化结果。
AI Orchestrator Prompt + JSON Schema 控制照片分析、画像生成、推荐、教程、聊天。
产品数据 本地 JSON → Supabase 先维护 30 条种子产品，后续扩展为数据库。
部署 Vercel / Render / Railway 确保可访问、可演示、有 fallback。

06 · 种子产品库

产品分类建议

洗发水、护发素、发膜、免洗护发、弹力素 / 卷发霜、 啫喱 / 摩丝、精油 / 护发油。

每类 4-6 个 总量 30 左右 先手工整理

单条产品字段

产品名、品牌、类别、价格层级、适合问题、不适合情况、质地、 使用方法、推荐理由模板。

07 · 其他注意事项

问题 简要方案
首页案例真实性 首版可使用模拟案例，但需要明确为“示例案例”；后续内测后替换为真实用户授权案例。
照片隐私 上传页说明照片仅用于生成护理建议；MVP 阶段可默认不长期保存原图。
AI 输出不稳定 所有 AI 结果使用结构化 JSON；前端只渲染固定字段，避免整页被自由文本破坏。
推荐显得不可信 每个产品类型和示例产品都必须附带“为什么适合你”和“怎么使用”。
产品数据不足 优先推荐产品类型，再给少量示例产品；不要把首版做成完整商品库。
护理建议过度承诺 文案中避免“保证有效”“专业诊断”等表达，统一写成“护理参考建议”。
用户不知道如何继续 结果页底部提供快捷追问，例如“预算更低怎么办”“不想用啫喱怎么办”“第二天炸毛怎么办”。
